# site colophon

this is the part where i tell you how the sausage gets made.

## the stack

this site is built with [next.js](https://nextjs.org/) and [react](https://react.dev/), styled with [tailwind css](https://tailwindcss.com/), and all the content is written in [mdx](https://mdxjs.com/). i use [bun](https://bun.sh/) as my runtime because it's fast and i'm impatient. code quality is handled by [biome](https://biomejs.dev/) which does linting and formatting in one tool, no arguing with prettier configs.

## writing & content

all the content on this site lives as mdx files in a `content/` directory. posts, snippets, projects, resources, logs, pages, they're all just markdown with some frontmatter at the top. no cms, no database, just files. i like it that way. it's simple, it's version controlled, and i can write in my text editor.

## hosting & deployment

the site is hosted on [vercel](https://vercel.com/). i push to main and it deploys. that's it. that's the workflow.

## design

i designed this site myself. the vibe is "what if a real developer's personal site looked like it was made by someone who grew up on the web in the early 2000s but also knows what a design system is." the pixel fonts, the bouncing star dividers, the dotted borders — it's all intentional. i wanted it to feel like *mine*.

## tools i use

- **editor:** [neovim](https://neovim.io/) — i live in the terminal
- **terminal:** [wezterm](https://wezterm.org/index.html)
- **font (code):** victor mono
- **font (site):** a mix of pixel and serif fonts because i contain multitudes

## the spirit of the thing

i've been making websites since 2004. this one is the latest in a long line of personal sites, and it probably won't be the last. i rebuild it whenever i feel like it, which is often. if something looks broken, it might be intentional. or it might not be. either way, i'm probably already fixing it.
